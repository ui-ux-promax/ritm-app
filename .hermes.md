# Fashion Shop (RITM) — Next.js E-commerce

Hermes: при работе в этом репо следуй этим правилам.

## Стек
- **Next.js 15** (App Router) + **React 18** + **TypeScript**
- **Prisma** + **Neon** (serverless Postgres)
- **NextAuth v5** (beta) + @auth/prisma-adapter
- **Tailwind CSS** + **Radix UI** + **lucide-react**
- **YooKassa** для платежей, **Cloudinary** для медиа
- **Resend** + react-email для писем
- **Upstash Redis** для rate limiting
- **Sentry** для мониторинга
- **Vitest** для unit-тестов, **Playwright** для e2e

## Команды
- `npm run dev` — запуск dev-сервера
- `npm run build` — production-сборка (запускай перед коммитом)
- `npm run typecheck` — проверка типов (ВСЕГДА перед коммитом)
- `npm run lint` — линтер
- `npm test` — unit-тесты (vitest)
- `npm run e2e` — e2e-тесты (playwright)
- `npx prisma generate` — регенерация клиента после изменения schema.prisma

## Правила
- **Всегда** запускай `npm run typecheck` после изменений в коде
- **Всегда** запускай `npm run build` перед тем как объявить задачу完成
- Не коммить `.env.local`, `.env*.local` — они уже в `.gitignore`
- Используй существующие компоненты из `components/` — не дублируй
- Prisma-клиент импортируется из `lib/db` (один экземпляр)
- Стили — только Tailwind, не пиши инлайн-стили без необходимости
- Server Components по умолчанию; `"use client"` только когда нужен interactivity
- При изменении `prisma/schema.prisma` — запусти `npx prisma generate`
- Не тратит токены на длинные объяснения — пиши код, краткие комментарии
- Предпочитай `patch` для точечных правок, а не переписывание файла целиком

## Токен-экономия
- Не выводи большие файлы в чат — используй `read_file` с offset/limit
- Не делай лишних веб-поисков без необходимости
- Краткие ответы без воды
- Используй `execute_code` для батч-операций (несколько правок за раз)

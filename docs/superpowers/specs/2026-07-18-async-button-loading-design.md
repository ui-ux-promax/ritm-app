# Асинхронные состояния кнопок

## Цель

Сделать ожидание сетевых и серверных операций понятным и безопасным: во время запроса кнопка показывает спиннер из дизайн-системы, визуально приглушается и не принимает повторный клик.

## Границы

Покрываются только элементы, которые инициируют асинхронную операцию: оформление заказа, вход и регистрация, добавление товара в корзину, изменение избранного, изменение состава корзины, формы профиля и асинхронные действия админки.

Не покрываются элементы, заблокированные по бизнес-правилу или навигации: отсутствующий размер, конец пагинации, статически недоступные действия и переключатели локального состояния.

## Подход

1. Повторно использовать `components/ui/button.tsx` и `components/admin/ui/button.tsx`, которые уже принимают `loading`, отключают кнопку и показывают `Loader2`.
2. Для нативных кнопок с существующим `isSubmitting`, `pending`, `adding` или аналогичным состоянием добавить такой же спиннер и `disabled`.
3. Во время загрузки сохранять геометрию кнопки, убирать обычную подпись, добавлять доступное имя/статус и применять тусклое disabled-состояние.
4. Не менять логику запросов, обработку ошибок, успешные состояния или тексты вне периода загрузки.

## Проверка

Добавить или расширить модульные тесты для затронутых компонентов, чтобы они проверяли отображение спиннера и блокировку повторного действия. Выполнить целевые тесты, `npm run typecheck` и полный набор тестов, если он укладывается в среду.
